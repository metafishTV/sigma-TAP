## Project Significance

DeLanda's reconstruction of assemblage theory is the most architecturally significant source yet distilled for sigma-TAP, because it provides the clearest articulation of how D&G's ontological framework maps to a computational modelling paradigm. The existing D&G ATP Ch15 distillation identified the tetravalent assemblage = L-matrix correspondence as an independent convergence; DeLanda's book provides the systematic justification for that correspondence and extends it in three directions: (1) parametrization of the L-matrix channels as continuously variable rather than categorical, (2) explicit connection between assemblage dynamics and the virtual/actual distinction that grounds TAP's adjacent possible, and (3) nested composition of assemblages that mirrors sigma-TAP's multi-scale agent architecture.

## Integration Points

- **Parametrized L-matrix as assemblage diagram**: DeLanda's most important contribution to sigma-TAP is the parametrization principle: the L-matrix's four channels should be understood as continuously variable parameters, not categorical slots. This suggests computing derived scores — a Territorialization score (L11/L21 ratio) and a Coding score (L12/L22 ratio) — that locate each agent in a 2D parameter space. The movement of an agent through this space over time would constitute its assemblage dynamics. This extends the current per-step L-matrix ledger into a trajectory analysis.
  - **Concept map entry**: `DeLanda_AssemblageTheory:parametrized_assemblage` → L_matrix_parametrization, T_score_C_score_derived_metrics

- **Properties vs capacities → type_set vs adjacent possible**: DeLanda's distinction maps precisely to sigma-TAP's existing architecture but reveals an undertheorized gap: the type_set records properties (actual), but the adjacent possible records capacities (virtual), and these require different formal treatment. Properties are intrinsic to the agent; capacities are relational — they depend on what OTHER agents the agent could interact with. Current sigma-TAP treats both as type counts. DeLanda suggests the adjacent possible should be formalized as an interaction-dependent structure, not a combinatorial count.
  - **Concept map entry**: `DeLanda_AssemblageTheory:properties_vs_capacities` → type_set_actual_vs_adjacent_possible_virtual

- **Intensive/extensive distinction → σ-field as intensive variable**: The σ-field operates as an intensive quantity: it drives qualitative transitions at critical thresholds (σ_critical), cannot be meaningfully averaged across agents, and produces novelty through differences rather than aggregation. M_t (type count) is extensive. This distinction, not currently formalized in sigma-TAP, could organize the variable taxonomy: extensive variables (M_t, N_agents, type_set sizes) vs intensive variables (σ, μ, Youn ratio).
  - **Concept map entry**: `DeLanda_AssemblageTheory:intensive_extensive` → sigma_field_as_intensive, extensive_vs_intensive_variable_taxonomy

- **Virtual diagrams → TAP equation as structured possibility space**: The TAP equation defines a state space with specific attractors (blow-up, collapse, steady-state). The parameter space (α, a, μ) defines a control space with specific bifurcations (regime transitions). DeLanda's framework names this existing structure and suggests formalizing the control space as a separate analytical object — the sensitivity sweeps already explore it, but without the theoretical framing.
  - **Concept map entry**: `DeLanda_AssemblageTheory:virtual_diagram` → TAP_state_space_attractors, parameter_space_as_control_space

- **Divergent actualization → multiple growth regimes from same equation**: The same virtual structure (TAP equation) admits multiple qualitatively different actualizations (growth regimes). The σ-field's role is precisely to navigate between these divergent solutions — selecting for metastable growth over blow-up or collapse. DeLanda's framework provides the ontological name for what the simulator already does.
  - **Concept map entry**: `DeLanda_AssemblageTheory:divergent_actualization` → TAP_growth_regimes, sigma_field_as_actualization_navigator

- **Nested assemblages → multi-scale agent architecture**: Stage 3B's family groups, junction adjacency, and community-level metrics already implement nested assemblages. DeLanda's framework provides principled justification: each level is a genuine individual entity with its own emergent properties, not just a statistical summary. This supports the architectural decision to model at multiple scales and suggests formalizing the inter-level causal mechanisms (bottom-up emergence + top-down constraint).
  - **Concept map entry**: `DeLanda_AssemblageTheory:nested_assemblages` → multi_scale_TAP, family_groups_as_assemblages, bottom_up_top_down_causality

- **Problems vs solutions → σ-field as one solution among many**: The sigma-TAP project's core problem (how to sustain innovation without extinction instability) has objective structure independent of any particular solution. The σ-field is ONE solution; other feedback mechanisms could address the same problem differently. This frames the project's research question ontologically and opens the door to comparing alternative σ-implementations.
  - **Concept map entry**: `DeLanda_AssemblageTheory:problems_vs_solutions` → sigma_TAP_problem_structure, sigma_field_as_one_solution

## Open Questions

1. **Parametrized L-matrix scores**: Should sigma-TAP compute derived T-scores (Territorialization = L11/L21 ratio) and C-scores (Coding = L12/L22 ratio) for each agent at each timestep? This would create agent trajectories in a 2D parameter space, enabling visualization of how agents move between assemblage configurations over time. The question is whether this adds analytical power beyond the raw L-matrix ledger.

2. **Properties vs capacities formalization**: DeLanda argues that capacities are relational (they depend on interaction partners) while properties are intrinsic. Current sigma-TAP treats the adjacent possible as a combinatorial count (how many new types COULD be generated). Should it instead model the adjacent possible as an interaction-dependent structure — different for each pair of agents, not a property of the agent alone?

3. **Intensive/extensive variable taxonomy**: Should sigma-TAP formally classify its variables into intensive (σ, μ, Youn ratio — subject to thresholds, non-additive) and extensive (M_t, N_agents, type_set sizes — additive, divisible)? This would organize the mathematical treatment and potentially reveal which variables drive qualitative transitions vs quantitative growth.

4. **TAPS as symmetry-breaking cascade**: DeLanda's symmetry-breaking framework suggests reading TAPS (Transvolution, Anopression, Praxis, Syntegration) as a sequential cascade rather than four parallel modes. Is this reading justified by the project's theoretical architecture, or does it impose a linearity that TAPS is designed to avoid?

5. **Plane of consistency as system diagnostic**: Could the distance from the plane of consistency (maximal deterritorialization + decoding) serve as a system-level diagnostic? A scalar measuring how far the total agent population is from full interconnection and full adaptive flexibility. This would be a macro-level metric complementing the per-agent σ values.
